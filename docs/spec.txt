一 系统总体目标
  本系统旨在针对北京交通大学学活食堂高峰时段拥挤、等座严重的问题，构建一套基
于离散事件仿真的就餐动线仿真系统。通过模拟学生从进入食堂、排队选餐、就餐到离开
的全过程，量化分析不同排队策略与空间布局对就餐效率的影响，为食堂管理者优化排队
策略与空间布局、提升就餐效率提供数据支撑与决策参考。具体目标如下：
  客流动态生成：基于非齐次泊松过程模拟学生到达，考虑课表时间（早中晚餐）、教学
楼分布、学生行为类型（急迫型、悠闲型、结伴型、错峰型）及入口方向影响。
  多队列排队策略对比：支持“单队多窗口”（学活一楼风格）、“多队多窗口”（学活二
楼风格）及可配置混合模式，实现基于剩余等待时间预估的贪心换队决策。
  餐桌流转与动线模拟：模拟餐桌分区布局（A 区近取餐口、B 区安静区、C 区大桌区），
根据学生偏好分配座位，处理结伴学生集体就座需求，通过等座队列间接反映动线冲突。
  数据统计与可视化分析：记录每位学生的排队等待时间、等座等待时间、总逗留时间，
定期采样 队列长度与餐桌 占用率，支持 一次运行对比多 种策略组合，自 动生成对比报告
（CSV 及图表）。

二 系统开发架构
  本系统采用 C/S（客户端/服务器）架构，但以本地应用形式运行，无远程服务端。架
构分为两层：
  前端（客户端）：负责参数配置、仿真运行控制、实时状态可视化（可选）及结果展示。
初期采用控制台+配置文件形式，中期可根据进度扩展为 Qt 图形界面。
  后端（仿真引擎）：负责核心仿真逻辑，包括事件调度、队列管理、餐桌资源管理、数
据统计等。后端作为独立模块被前端调用。

4.2 模块协同与数据交换
  事件驱动：所有模块通过全局事件队列（std::priority_queue<Event*>）交互。每个事件
（如到达、服务开始、就餐结束）触发相应模块处理函数，并可能产生新事件。
  采样回调：数据统计模块注册回调函数，在固定仿真步长（如 10 秒）被触发，读取各
模块状态（队列长度、餐桌占用率）并记录。
  学生对象传递：学生对象指针在整个仿真流程中传递，各模块在学生属性中记录相关
时间戳（如排队开始时间、服务开始时间等）。

五、软件原型设计
  原型设计用于在编码前快速构建系统界面框架、交互逻辑和视觉效果，降低沟通成本，
减少开发偏差。本项目提供三种主流原型设计途径，团队可根据协作需求选择其一或混合
使用。
5.1 途径一：墨刀（快速原型，在线协作）
  特点：操作简单，内置大量组件库，支持手机/网页原型，适合快速出稿。提供网页版
和客户端，支持多人实时协作、评论、分享预览。
  适用场景：初期快速勾勒界面布局、简单跳转逻辑，适合非高保真需求。
  使用方式：1.注册墨刀账号，创建项目（可选择“网页”或“自定义尺寸”）。2.拖拽组
件（按钮、表格、弹窗、图表占位符）到画布，设置交互（点击跳转、弹窗）。3.通过“预
览”按钮在浏览器中查看效果，或生成二维码在手机端演示。
  输出：可生成在线演示链接、下载离线 HTML 包、导出图片或 PDF。
5.2 途径二：Pixso / 即时设计（高保真协同设计）
  特点：国内主流在线设计工具，功能类似 Figma，支持矢量编辑、组件复用、自动布
局、实时协作。两者功能相近，团队可任选其一。
  适用场景：制作高保真原型，精细控制界面元素、颜色、字体，适合后期需要真实界
面视觉的演示。
  使用方式：注册账号，创建文件。利用矩形、文本、图标等绘制界面，或从资源库导
入预制组件。使用“原型”模式添加交互（点击、悬停、拖拽等），设置过渡动画。点击“演
示”按钮进入演示模式，模拟真实操作。
  输出：可导出设计稿图片、生成可交互的链接、下载开发标注（CSS/代码）。
5.3 途径三：Axure RP（专业交互原型）
  特点：功能强大，支持条件逻辑、变量、动态面板等复杂交互，适合制作高度模拟真
实系统的原型。
  适用场景：需要演示复杂逻辑（如参数配置、多步骤仿真控制、动态图表更新）的场
景，或用于用户测试。
  使用方式：
  创建项目，绘制界面元素（按钮、输入框、中继器等）。
  通过“交互”面板设置事件（如单击时显示/隐藏面板、设置文本、触发全局变量）。
  使用“中继器”模拟表格数据展示，使用“动态面板”实现多状态切换。
  按 F5 生成 HTML，在浏览器中运行。
  输出：生成可独立运行的 HTML 文件夹，包含完整交互逻辑。
5.4 原型内容规划
  无论选择哪种工具，原型应至少包含以下界面：
  参数配置界面：
  窗口设置（数量、类型、服务时间分布）
  餐桌设置（分区数量、座位数、偏好权重）
  学生设置（到达率曲线、类型比例、结伴参数、换队阈值）
  排队策略选择（单队/多队/混合模式）
  仿真时长、随机种子、输出路径
  仿真监控界面（可选）：
  动态显示各窗口队列长度（柱状图或数字）
  动态显示各餐桌分区占用情况（色块）
  实时输出当前时间、已完成事件数
  控制按钮：开始、暂停、重置
  结果展示界面：
  统计指标卡片（平均排队时间、等座时间、总时间等）
  图表缩略图（等待时间分布、队列长度时序等）
  导出数据按钮（CSV、图表图片）
  帮助/关于界面：简要说明系统用途、操作指南。
六、图表生成途径
仿真系统输出 CSV 数据后，需要生成可视化图表用于分析和报告。提供多种图表生成途径，
可根据团队熟悉度和需求选择。
6.1 途径一：Python Matplotlib（基础绘图）
   特点：Python 最经典的绘图库，功能全面，可生成线图、柱状图、直方图、箱线图、
散点图等。支持自定义样式、中文显示。
   使用方式：
   安装 Python（≥3.6）及 Matplotlib、Pandas、Numpy。
   编写脚本读取 CSV 文件（如 students.csv、queue_sample.csv、table_sample.csv）。
   使用 Matplotlib 绘制：
   排队等待时间分布直方图（plt.hist）
   队列长度随时间变化曲线（plt.plot）
   餐桌占用率时序图（plt.plot）
   不同类型学生等待时间箱线图（plt.boxplot）
   保存为 PNG、PDF 或 SVG。
   优点：免费、灵活、代码可复用；缺点：需要一定的 Python 基础。
6.2 途径二：Python Seaborn（统计图形）
   特点：基于 Matplotlib 的高级接口，内置美观的统计图形，如分布图、热力图、分类
图等，适合快速生成专业图表。
   使用方式：
   安装 Seaborn：pip install seaborn
   使 用 seaborn.histplot 绘 制 等 待 时 间 分 布 ， seaborn.boxplot 绘 制 分 组 箱 线 图 ，
seaborn.lineplot 绘制时序曲线。
   优点：默认样式美观，代码简洁；缺点：对于复杂定制可能需要结合 Matplotlib。
6.3 途径三：Plotly（交互式图表）
   特点：可生成交互式网页图表，支持缩放、悬停提示、数据点选择，适合演示和动态
探索。
   使用方式：
   安装 Plotly：pip install plotly
   使用 plotly.express.line、plotly.express.histogram 等生成图表，保存为 HTML 文件。
   可在浏览器中打开，与图表交互。
   优点：交互性强，适合演示；缺点：生成的文件较大，需要浏览器支持。
6.4 途径四：Excel / WPS（手动绘图）
   特点：无需编程，直接利用办公软件导入 CSV 数据，通过图表向导生成图表。
   使用方式：
   将 CSV 文件用 Excel 打开，选中数据列。
   插入图表（折线图、柱状图等），调整格式。
   优点：门槛低，适合临时查看数据；缺点：不适合批量生成，样式一致性差。
6.5 途径五：在线数据可视化工具（如 Flourish、Datawrapper）
   特点：在线平台，上传数据后选择图表类型，可生成美观的交互图表，支持嵌入网页
或导出。
   使用方式：
   注册账号，上传 CSV 文件。
   配置图表类型、颜色、标签，生成链接或导出图片。
    优点：快速出图，无需编程；缺点：免费版有功能限制，数据需上传至云端。
6.6 图表生成流程设计
    为了在报告中统一图表风格，建议采用 Python + Matplotlib/Seaborn 方案，并编写一个
统一的绘图脚本 plot_results.py，实现以下功能：
    读取指定目录下的三个 CSV 文件。
    自动生成以下图表并保存为 PNG：
    等待时间分布对比图：急迫型、悠闲型、结伴型的排队等待时间直方图叠加，等座等
待时间直方图。
    队列长度时序图：各窗口队列长度随时间变化的折线图，突出高峰时段。
    餐桌占用率时序图：各分区占用率随时间变化的曲线。
    平均指标对比图：不同策略组合下的平均等待时间、等座时间、总时间柱状图（多策
略对比时）。
    将图表插入到最终报告（Word/PDF）中。

七、数据模型
7.1 输入数据（用户配置）
    仿真配置文件（config.json）：采用 JSON 格式存储所有可配置参数。示例结构详见《系
统设计规格说明书》第 6.1 节（已在前文给出）。配置内容包括仿真时长、随机种子、学生
到达率分段函数、类型比例、结伴规模分布、就餐时长范围、窗口列表、餐桌分区等。
7.2 运行过程数据（实时产生）
    学    生    对    象   ：    内    存    中    存   储    为     Student   结    构   体    ，    包    含
id,type,group_id,arrival_time,queue_start_time,service_start_time,eat_start_time,leave_time,wa
it_queue_time,wait_seat_time,total_time 等字段。
    队列长度采样：每固定仿真步长调用回调，记录时间、各窗口队列长度、总排队人数，
存入 std::vector<QueueSample>。
    餐桌占用采样：每固定仿真步长调用回调，记录时间、各分区占用数、总占用率，存
入 std::vector<TableSample>。
7.3 输出数据（仿真结束后）
    students.csv：每行一个学生，字段包括 ID、类型、组 ID、各阶段时间戳及计算指标。
    queue_sample.csv：每行一个采样时刻，字段包括时间、各窗口队列长度、总排队人数。
    table_sample.csv：每行一个采样时刻，字段包括时间、各分区占用数、总占用数、占
用率。
7.4 数据流转与存储
    输入：config.json 由主控模块加载，解析后分发给各模块初始化。
    运行：仿真过程中，各模块实时更新内存中的学生对象和采样数据结构。
    输出：仿真结束后，统计模块遍历所有学生对象和采样记录，写入 CSV 文件。
    图表生成：外部 Python 脚本读取 CSV，根据配置生成多种图表，保存为 PNG/PDF，
并可选生成 HTML 交互图表。

八、总结
    本系统设计规格说明书明确了就餐仿真系统的总体目标、技术架构、模块划分、交互
原型及数据模型，并针对原型设计和图表生成提供了多种可选途径，满足团队不同技术背
景和展示需求。后续开发将严格按照本设计进行，并保持文档与代码同步更新。
